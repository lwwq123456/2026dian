# 任务 3.3 实验记录：Gated DeltaNet 并行化与架构优化

## 1. 实验内容
* **全矩阵并行化**：采用 **WY 表示法**，利用 `torch.linalg.solve_triangular` 将 $O(L)$ 的时序递推折叠为 $O(1)$ 的张量运算。
* **引入位置编码**：在视觉序列输入端注入可学习的位置编码（`pos_embed`），弥补展平操作丢失的 2D 拓扑信息。

## 3. 探索与思考

* **并行计算中数学公式的学习**
由于在Fashion-Mnist数据集中每张图片只有28*28的大小，并且每张图片之间并无关联，所以在该数据集上训练之时并没必要对每张图片进行分块，直接将每一张图片作为一个整体进行并行训练即可。只需让原论文最终给出的并行计算公式中的S_t=0，就可以得到和以下推导所得出的一样的在Fashion-Mnist数据集上的并行加速公式。

### 1. 提取“更新向量” 
原始的隐状态递推公式中，包含了繁杂的历史状态相乘。将其整理，并把当前时刻**“真正需要吸收的新知识”**定义为更新向量 $\bar{u}_t$：
$$\bar{u}_t = \beta_t (v_t - \alpha_t S_{t-1} k_t)$$

代入原式后，状态更新简化为：
$$S_t = \alpha_t S_{t-1} + \bar{u}_t k_t^T$$

### 2. 构造下三角方程组 
通过将上述简化的 $S_t$ 无限向前展开，并代回 $\bar{u}_t$ 的定义式，发现：**当前时刻的 $\bar{u}_t$，只依赖于 $v_t$ 和所有过去的 $\bar{u}_{j<t}$。**

这意味着它构成了一个**严格下三角方程组**：
$$(I + M) \bar{U} = \beta V$$
*其中 $M$ 是由时间衰减 $\Gamma$ 和键值内积 $K K^T$ 构成的严格下三角矩阵。*

### 3. 公式与代码

**第一步：计算全局门控衰减矩阵 $\Gamma$**
$$\Gamma_{ij} = \prod_{m=j+1}^i \alpha_m$$
```python
cum_log_alpha = torch.cumsum(torch.log(alpha.squeeze(-1) + 1e-8), dim=1)
Gamma = torch.tril(torch.exp(cum_log_alpha.unsqueeze(2) - cum_log_alpha.unsqueeze(1)))
```

**第二步：构造严格下三角系统矩阵 $M$**
$$M_{ij} = \beta_i \Gamma_{ij} (k_i^T k_j) \quad (i > j)$$
```python
M = torch.tril(beta * (Gamma * torch.bmm(k, k.transpose(1, 2))), diagonal=-1)
```

**第三步：调用底层算子，极速求解 $\bar{U}$**
$$\bar{U} = (I + M)^{-1} (\beta V)$$
```python
U_bar = torch.linalg.solve_triangular(I + M, beta * v, upper=False)
```

**第四步：结合掩码，计算最终输出 $O$**
$$O = (\Gamma \odot (Q K^T)) \bar{U}$$
```python
attn = torch.tril(Gamma * torch.bmm(q, k.transpose(1, 2)))
gdn_out = torch.bmm(attn, U_bar)
```

* **精度 vs 速度**
  实验发现，经过极其复杂的并行优化，3.3 的准确率反而微降了 0.41%。经过底层推导，我确认并行算法与串行算法在数学上绝对等价，可能是统计学上的随机波动。从损失值看，训练15轮后3.3的损失值更大，说明加入位置编码后可能提高了模型的学习难度。但在真实的系统工程中，用 0.3% 的微小精度损耗换取 3 倍的训练提速，是完全值得的。

* **防过拟合的控制**
  将训练严格锚定在 15 轮，使学习率在末期精准降至谷底。若盲目增加轮数，模型会在 Train Loss 降至 0.1 左右时陷入过拟合的陷阱。
